智能网联车纵向编队控制 · 第二篇论文

基于实时通信延迟的自适应编队控制框架

动态时距（DTH）+ 自适应TubeMPC + 事件触发协同设计

背景与问题

CTH 局限 传统固定时距策略（CTH）中，时距 h 为常数，期望间距仅随自车速度线性变化。在V2X网络中，通信延迟实时波动，控制器基于过期状态信息决策，固定间距无法反映信息质量的下降，延迟越大安全风险越高。

// CTH（第一篇）
d_ref = d₀ + h · v(t)
// h = 0.6 s 固定常数，与通信质量无关
DTH 设计

创新 将V2X实测单向通信延迟 τ(t) 实时映射至时距 h(t)，延迟越大期望间距越保守。信息可信度直接影响安全裕量，这在V2X编队控制领域鲜有先例。

// DTH（第二篇创新点1）
h(t) = h₀ + γ · τ(t)
h(t) ∈ [h_min, h_max] = [0.6, 1.5] s

d_ref,i(t) = d₀ + h(t) · v_i(t)
e_s,i(t) = (s_{i-1} − s_i) − d_ref,i(t)

// 参数: h₀=0.8s, γ=3.3, d₀=1.5m
// τ(t): V2X单向延迟 = t_recv − t_send

背景与问题

MPC 局限 标准MPC假设系统无扰动（w=0），但实车中执行器输出与期望加速度存在偏差。实车数据标定扰动上界 |w| ≤ 0.5 m/s²。扰动累积可能导致实际状态越出约束边界，鲁棒性不足。

// 标准MPC（忽略扰动）
x(k+1) = Aₐ x(k) + Bₐ u(k)
// 隐含假设 w = 0，实车中不成立
TubeMPC 设计

创新 将系统分解为标称系统与误差系统，用Tube包住扰动影响的边界。Tube大小随实时通信延迟自适应调整，延迟越大Tube扩张，约束收紧更多，鲁棒性增强。

// 实际系统
x(k+1) = Aₐ x(k) + Bₐ u(k) + w(k), |w| ≤ 0.5

// 分解：标称 + 误差
标称: x̄(k+1) = Aₐ x̄(k) + Bₐ ū(k) ← MPC在此求解
误差: e(k+1) = Aₐ e(k) + w(k), e = x − x̄

// 自适应Tube（创新）
Ω(t) = Ω₀ + β · τ(t)

// 约束收紧
x_min + Ω(t) ≤ x̄(k) ≤ x_max − Ω(t)

// 实际控制量
u = ū + K · e

背景与问题

周期控制局限 周期性控制每个采样时刻都求解MPC并进行V2X通信。当编队状态变化缓慢时，存在大量不必要的计算与通信开销，浪费车载计算资源和V2X信道带宽。

事件触发设计

创新 以Tube余量实时调节触发阈值，误差逼近Tube边界时才触发MPC求解与V2X通信，否则保持上次控制量。三个模块统一由通信延迟驱动，形成闭环协同框架。

// 触发函数
f(t) = ‖e_i(t)‖ − σ · ‖Ω(t)‖

// 触发条件
f(t) > 0 → 触发：重新求解MPC + V2X通信
f(t) ≤ 0 → 不触发：保持上次控制量 u_last

// Tube余量
ρ(t) = 1 − ‖e‖ / ‖Ω‖, ρ ∈ [0, 1]
闭环协同逻辑

三个创新点统一由 τ(t) 驱动，形成完整闭环：

τ(t) ↑ → h(t) ↑　期望间距增大 DTH · 创新点1
τ(t) ↑ → Ω(t) ↑　Tube扩张，鲁棒性增强 TubeMPC · 创新点2
τ(t) ↑ → σ·‖Ω‖ ↑　触发阈值增大，更敏感 事件触发 · 创新点3
